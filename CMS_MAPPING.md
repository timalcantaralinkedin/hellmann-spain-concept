# Drupal CMS Build Map — Hellmann Spain Landing Page

This document maps every section of the concept page to a **Drupal Paragraph** in Hellmann's CMS. Use it as a build sheet inside the Drupal admin UI.

## How Hellmann's pages are assembled (observed from live site)

Every section follows this nesting pattern, visible in the HTML source of `hellmann.com/en/belgium-netherlands-luxembourg`:

```
paragraph--section            (the outer band — controls vertical spacing)
└─ paragraph--grid container  (12-column grid wrapper, max-width 1280px)
   └─ flex flex-wrap c-grid   (Tailwind row)
      └─ paragraph--column    (12-col span: w-full ml-0, md:w-7/12 ml-0, md:w-4/12 ml-1/12, etc.)
         └─ paragraph--[type] (the actual content component)
```

The CMS editor's job is to add a **paragraph--section**, place one or more **paragraph--column** items inside it (each with a column-span class), and then drop content components (text, headline, image, block reference, facts, etc.) into each column.

## Section-by-section build map

### 1. Hero — `<section class="hero --full-image">`

Not a paragraph. The hero is a **node field** (`field--field-key-image` for the background, `field--field-headline` for H1, `field--field-text` for the lede). Editor sets:
- **Hero image** → Media library → image (style: `hwl_key_image_hero_2x`, 1180×780)
- **Headline** → "Hellmann Spain"
- **Lede** → 2 sentences
- **CTAs** → Two `.button.--frosty` links (Let's Do Business + Find a Local Office)
- **Breadcrumb** → auto-generated by Drupal from page hierarchy

### 2. Welcome / Intro — 1 section, 2 columns

| Column | Span | Components |
|---|---|---|
| Left | `md:w-7/12 w-full ml-0` | `paragraph--text` → field-headline + field-text (4 paragraphs) |
| Right | `md:w-4/12 w-full ml-1/12` | `paragraph--block` → reference **person** block (Country Manager Spain), then `paragraph--block` → reference **Find your Local Office** callout block |

**Pre-built blocks needed in Drupal:**
- `block-contact` for the Country Manager (`.person.--teaser` markup)
- `block-find-office` for the callout CTA

### 3. Facts band — 1 section, 1 column, 1 facts component

| | |
|---|---|
| Section | `paragraph--section mt-lg` |
| Column | `w-full ml-0` |
| Component | `paragraph--facts --appearance-framed_big` with 4 facts items |

**Each fact item fields:** number, label, sub-label. Use `framed_big` appearance variant (one of: clean / big / framed / framed_big).

### 4. Our Offices (Locations) — 1 section, 1 headline + 4 columns

| Column | Span | Components |
|---|---|---|
| Headline row | `w-full ml-0` | `paragraph--headline` → "Four locations, four specialisations." (with eyebrow) |
| Card 1 | `md:w-3/12 w-full` | `paragraph--related_content` → Barcelona card (image + title + body) |
| Card 2 | `md:w-3/12 w-full` | `paragraph--related_content` → Madrid card |
| Card 3 | `md:w-3/12 w-full` | `paragraph--related_content` → Bilbao card |
| Card 4 | `md:w-3/12 w-full` | `paragraph--related_content` → Valencia card |

**Content per card:** background image (style: `landscape` crop), city name (h3), one-line specialisation. Hellmann does not have a dedicated "location-card" component — repurpose `paragraph--related_content` with image and short body.

### 5. Our Products & Services — 1 section, headline + 4 product cards

| Column | Span | Components |
|---|---|---|
| Headline | `w-full ml-0` | `paragraph--headline` → "Four modes. One coordinated network." |
| Card 1–4 | `md:w-3/12 w-full` each | `paragraph--related_content` → image + h3 + bulleted list + "Learn more" link |

Use the **same `paragraph--related_content` component** as the Benelux page uses for its products grid. Bullet list goes in the body via Drupal's rich-text editor (use the list-arrow appearance to get the chevron bullets).

### 6. Industry Solutions — 1 section, headline + 4 text cards (background-tinted)

| Column | Span | Components |
|---|---|---|
| Section | `paragraph--section` inside a `bg-rlight` wrapper (or use section background-color setting) |
| Headline | `w-full ml-0` | `paragraph--headline` → "Built around the industries that shape Spain." |
| Card 1–4 | `md:w-3/12 w-full` each | `paragraph--text` (text-only — no image), or `paragraph--related_content` with image hidden |

This matches Benelux's "Our Industry Solutions — Expertise Across Sectors" — text-only cards in a tinted band.

### 7. Spotlight feature (Iberia ↔ LATAM) — 1 section, 2 columns side-by-side

| Column | Span | Components |
|---|---|---|
| Text | `md:w-6/12 w-full` | `paragraph--text` → headline + body + bulleted list + `paragraph--cta` (button) |
| Image | `md:w-6/12 w-full` | `paragraph--image` OR `paragraph--media-assets` (for image gallery slider) |

This matches the Benelux "New Airfreight Consolidation Service: Benelux → Dubai" pattern exactly.

### 8. Contact CTA band — 1 section (blue background), 2 columns

| Column | Span | Components |
|---|---|---|
| Section | `paragraph--section` with blue background variant |
| Text | `md:w-7/12 w-full ml-0` | `paragraph--text` (white text variant) — "Let's do business in Spain." |
| Button | `md:w-4/12 w-full ml-1/12` | `paragraph--cta` → "Get in touch" linked to mailto |

### 9. Vision / Quote — 1 section, full-width quote block

| | |
|---|---|
| Component | `paragraph--quote_vision` |
| Fields | quote (h2), body text, optional values list (caring, entrepreneurial, etc.) |

Hellmann's site already has a "Hellmann Promise" component — this slots into it directly.

### 10. Footer

Footer is a **global region**, not part of the page content. Hellmann's footer auto-renders on every page. Editor does not touch it per-page.

## Drupal Paragraph types used (summary)

| Paragraph type | Used in sections |
|---|---|
| `paragraph--section` | Every section (wrapper) |
| `paragraph--grid` | Every section (12-col wrapper) |
| `paragraph--column` | Every column |
| `paragraph--text` | Intro, industry cards, feature text, contact CTA |
| `paragraph--headline` | Every section header |
| `paragraph--image` | Feature visual |
| `paragraph--block` | Person card, Find your Office callout |
| `paragraph--facts` (`--appearance-framed_big`) | Stats band |
| `paragraph--related_content` | Location cards, product cards |
| `paragraph--cta` | Contact band button, feature LATAM button |
| `paragraph--quote_vision` | Vision section |
| `paragraph--media-assets` (optional) | Feature image gallery (if using carousel) |

## Pre-built blocks to create once and reference everywhere

1. **`block-contact-spain`** — Country Manager Spain person card (image + name + position + email link)
2. **`block-find-office-spain`** — "Find your Local Office" callout with link to offices anchor or office finder page

These appear in the right column of the intro and could be reused on other Spain sub-pages.

## Build order (recommended in Drupal admin)

1. Create the country page node ("Spain") with the hero key image
2. Create the two reusable blocks (above)
3. Add paragraphs to the page body in this order: intro → facts → offices → products → industries → spotlight feature → contact band → vision
4. Set each section's spacing (`mt-lg` modifier or section padding setting)
5. Preview, adjust column widths if anything wraps wrong, publish

## Column-width cheat sheet

Hellmann's grid uses the same Tailwind-style classes:

| Visual layout | Classes |
|---|---|
| Full width | `w-full ml-0` |
| Half + half | `md:w-6/12 w-full` × 2 |
| 7 + 4 with gap | `md:w-7/12 w-full ml-0` + `md:w-4/12 w-full ml-1/12` |
| 4 equal columns | `md:w-3/12 w-full` × 4 |
| 3 equal columns | `md:w-4/12 w-full` × 3 |
| Centered 8/12 | `md:w-8/12 w-full ml-2/12` |

Mobile: every column collapses to `w-full` automatically (their CSS handles the breakpoint).
